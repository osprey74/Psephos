# Psephos アプリ枠（chrome）デザイン — Claude Design 向け Handoff

> このドキュメントを **claude.ai/design** に渡し、Psephos（ClockworkPi PicoCalc 向けプログラマブル関数電卓）の **画面装飾レイヤ（chrome）** をデザインしてもらうための指示書です。
>
> ヘルプ画面の Handoff（[claude_design_handoff_help.md](claude_design_handoff_help.md)）と同じ「セマンティック 4 色」設計思想を踏襲します。

---

## 1. 出力に求めるもの

- **1 枚の PNG 画像**（`chrome.png`）
- **320 × 320 ピクセル ちょうど**
- セマンティック 4 色のみ使用（次節 §3）
- **画面中央の動的領域は背景色（BG）で塗りつぶす**（§4 で詳述）

---

## 2. ターゲットデバイス（ヘルプ画面と共通）

| 項目 | 値 / 制限 |
|---|---|
| 解像度 | **320 × 320 ピクセル** ちょうど |
| 色深度 | 4bit / 16 色だが、本画像は **セマンティック 4 色** で設計 |
| ピクセル形状 | 正方ピクセル |
| 視聴距離 | 約 30 cm 手持ち |
| アンチエイリアス | 不可（4 色量子化で潰れます） |

---

## 3. セマンティック 4 色（必須）

| 役割 | RGB | 用途 |
|---|---|---|
| **BG** | `#000000`（黒） | 背景・動的領域 |
| **FG** | `#FFFFFF`（白） | 主テキスト |
| **DIM** | `#808080`（中間灰） | 補助情報・サブテキスト |
| **ACC** | `#FFC800`（金黄色） | ロゴ・装飾・強調 |

実機側でテーマ変更時に自動色置換されます。

---

## 4. レイアウト構成（最重要）

画面 320×320 は **3 領域** に分割します：

```
┌─────────────────────────────────────┐ y=0
│   chrome 上部 (16 px / 2 行)        │
│   ヘッダー: PSEPHOS - Programmable scientific calculator │
├─────────────────────────────────────┤ y=16
│                                       │
│                                       │
│         動的領域 (296 px / 37 行)    │
│         BG 色で塗りつぶし            │
│         ※実機がここに履歴・入力を    │
│           リアルタイム描画            │
│                                       │
│                                       │
├─────────────────────────────────────┤ y=312
│   chrome 下部 (8 px / 1 行)         │
│   ステータスバー                      │
└─────────────────────────────────────┘ y=320
```

### 重要な制約

- **y=16 から y=311 までの中央領域（296 px 高さ）は完全に BG 色（黒）で塗りつぶしてください**
  - この領域は実機がリアルタイムに上書き描画します
  - 何か描いても消されますので空白にしてください
- **y=0 〜 y=15 の上部 16 px** が chrome 上部
- **y=312 〜 y=319 の下部 8 px** が chrome 下部

---

## 5. 上部 16 px に載せる内容

### 必須

- **アプリ名: `PSEPHOS - Programmable scientific calculator`**
  - ASCII 部分（`PSEPHOS`）は ACC 色でロゴ的に強調
  - `- Programmable scientific calculator` 部分は FG または DIM で説明的に
  - 全体で 16 px 高さに収まる Sans Serif 系（英文なので Consolas / Inconsolata 等のモノスペースも可）
  - 推奨フォントサイズ: 10-12 px（英文を 1 行に収めるためやや小さめ）

### 推奨される追加要素（任意）

- 左右端の装飾的な括弧 `[` `]` や罫線（DIM か ACC で）
- 右端に小さなインジケータ（バージョン番号 `v0.4` 等）— DIM で控えめに
- 上下の罫線（DIM の 1 px 細線）で区切り感を出す

### NG

- 16 px を超えるはみ出し
- 中央テキストが動的領域にかかる

---

## 6. 下部 8 px ステータスバーに載せる内容

### スペースが極小（8 px 高さ）

入る情報は限られます。以下のいずれか / 組み合わせを推奨：

- 左寄せ: `READY` `IDLE` などの状態表記（5-6 文字）
- 中央: 区切り文字（`·` `─` `█` 等）
- 右寄せ: バージョン番号 `v0.4` または現在テーマ名（短縮形 OK）

### 推奨フォントサイズ

- 8 px ピクセルフォント風が望ましい（縦に収まりやすい）
- ASCII のみなら 6-8 px、日本語を入れるなら 8 px が下限

### 装飾

- 上端に薄い区切り線（DIM の 1 px）で動的領域から仕切る
- ACC や ACC アンダーラインで「機材っぽさ」を出すと CRT らしい

---

## 7. 装飾の方向性: レトロ CRT ターミナル

総司様の指定です。以下の要素を取り入れてください：

### あり

- **罫線文字** `─ │ ┌ ┐ └ ┘ ┼ ╋` などで枠線を構成
- **モノスペース風のフォント**（特に ASCII 部分）
- **角張った装飾**: 円や曲線より直線・直角
- **ハードな配色**: グラデーション禁止、塗り分けはくっきり
- 任意で **ピクセル化された矢印・ドット**

### なし

- グラデーション、ぼかし、影
- ラウンド角（直角でお願いします）
- アイコン的なエフェクト（光沢・反射等）
- 5 色目以降の色

### 参考イメージ

- 1980 年代のテキスト端末（VT100, DEC ターミナル）
- レトロ電卓 / シンセサイザーの LCD パネル
- BBS / 黒画面コマンドプロンプト
- フィルム調 CRT モニター（角張った見た目で OK、走査線エフェクトは不要）

---

## 8. デザインの自由度

以下は Claude Design / 総司様の裁量にお任せします：

- 上部・下部の枠線スタイル（実線 / 二重線 / 装飾的 / 無し）
- ロゴ位置（左 / 中央 / 右寄せ）
- ステータスバーの具体的な文字列（READY / IDLE / 任意）
- ACC のアクセント箇所（ロゴ / 区切り / 両方）
- 全体の余白配分

---

## 9. 避けてほしいこと

- 320×320 以外のキャンバスサイズ
- §3 で指定した 4 色以外の使用
- 中央領域（y=16〜311）への描画
- アンチエイリアス
- 細すぎる線（1 px 線は LCD で見えにくい場合あり、2 px 推奨）
- 装飾過多（CRT は「機能美」が原則、シンプル＞華美）

---

## 10. エクスポート形式

- **PNG**、320 × 320 px ちょうど
- RGB モードまたはインデックスカラー（"P"）
- 使用色は §3 の 4 色のみ
- ファイル名: `chrome.png`
- 保存先: `g:\dev\Psephos\assets\chrome.png`

---
