# Psephos ヘルプ画面 — Claude Design 向け Handoff

> このドキュメントを **claude.ai/design** に渡し、Psephos（ClockworkPi PicoCalc 向け関数電卓）のヘルプ画面 2 ページ分のデザインを生成してもらうための指示書です。

---

## 1. 出力に求めるもの

- **2 枚の PNG 画像**（ページ 1 / ページ 2）
- **各画像 320 × 320 ピクセル ちょうど**
- **4 色だけを使用する「セマンティック 4 色」設計**（次節 §3 を必読）
- アンチエイリアスなし、または 4 値量子化に耐える線幅

## 2. ターゲットデバイスの制約

| 項目 | 値 / 制限 |
|---|---|
| 物理画面 | ILI9488 LCD、約 70 × 70 mm の正方形 |
| 解像度 | **320 × 320 ピクセル** ちょうど |
| 色深度 | 4 bit / 16 色だが、本画面は **1 bit（白黒）として設計** |
| ピクセル形状 | 正方ピクセル、1:1（縦横比歪みなし） |
| 文字レンダリング | デバイス側で 6×8 ASCII フォントが固定。日本語は本画像で持参 |
| 動作環境 | 室内手持ち、約 30 cm 視聴距離 |
| 操作系 | 物理 QWERTY キー（マウス・タッチなし） |

### 視認性の必須要件

- **日本語文字は最低でも 12 px 角**（読みやすさ重視なら 14–16 px）
- 細線（1 px）の縦線が連続するパターンは LCD 表示で見えにくい → 線幅は 2 px 以上を推奨
- 画面端から 4 px 以上の余白を確保（ベゼルに隠れる領域への配慮）
- **アンチエイリアスを 1-bit 閾値化すると痩せ細る**ため、フォントは少し太め（Medium〜Bold）を選ぶこと

## 3. テーマ・色設計（重要）

### セマンティック 4 色設計の理由

PicoCalc は実機側で **5 種類のカラーテーマ**（default / amber / green / cyan / invert）を切替えられます。テーマが変わってもヘルプ画像が自動追随するように、**画像内では「実色」ではなく「役割」を表現** していただきます。

### 使用してよい 4 色のみ

デザインには **以下 4 色だけ** を使用してください。他の色（中間色・グラデーション）を使うと変換時に誤割当されます。

| 役割 | デザイン上の色（RGB） | 用途 |
|---|---|---|
| **BG**（背景） | `#000000`（黒） | キャンバス全体の地、ベタ塗りエリア |
| **FG**（前景・本文） | `#FFFFFF`（白） | 通常の本文テキスト |
| **DIM**（弱め・補足） | `#808080`（中間灰） | サブテキスト、ページ番号、注釈、控えめなラベル |
| **ACC**（強調・アクセント） | `#FFC800`（金黄色） | セクション見出し、区切り線、ボタン枠、重要キーワード |

### テーマ切替時の自動マッピング

実機側は画像のピクセル値を「ロール」として読み取り、現在のテーマに応じて自動的に実色を割り当てます：

```
default テーマ:   BG=黒 / FG=白    / DIM=暗灰 / ACC=黄
amber テーマ:     BG=黒 / FG=黄    / DIM=暗黄 / ACC=白
green テーマ:     BG=黒 / FG=緑    / DIM=暗緑 / ACC=白
cyan テーマ:      BG=黒 / FG=シアン / DIM=暗シアン / ACC=白
invert テーマ:    BG=白 / FG=黒    / DIM=暗灰 / ACC=暗赤
```

### 守るべきこと

- 4 色「ちょうど」を使うこと（5 色目を入れない）
- アンチエイリアスはオフ。文字の縁に中間色が出ないように
- グラデーション・影・半透明は禁止
- 区切り線・罫線にも DIM か ACC のいずれかを選ぶ（中間色 NG）

### デザイナーへのヒント

- **本文は FG、控えめな注釈は DIM、見出しと区切りは ACC**、という階層を意識
- 全体の 70-80% は FG、10-20% は DIM、5-10% は ACC が目安
- ACC を使いすぎると「全部強調」になって視覚的階層が崩れます

## 4. レイアウトの世界観

- **レトロ CRT ターミナル / 科学電卓** の雰囲気
- 罫線（box-drawing 風）を控えめに使うのは可
- 装飾より「情報の読みやすさ」を優先
- ASCII 部分（関数名・記号）は等幅フォント風が望ましい
- 日本語部分は Sans Serif（Noto Sans JP, Yu Gothic 系）

## 5. ページ 1 に載せる内容

タイトル: **`Psephos -- 関数電卓 ヘルプ (1/2)`**

### 関数カテゴリ

| カテゴリ | 関数名（並び順は維持） |
|---|---|
| 三角 | `sin cos tan asin acos atan atan2` |
| 指数対数 | `exp log log10 sqrt pow` |
| 端数 | `floor ceil fabs abs round` |
| 角度変換 | `radians degrees` |
| 進数変換 | `hex bin oct int float` |
| 定数 | `pi e tau` |
| 汎用 | `min max` |

### 直近結果の参照

- `ans` — 最新の計算結果
- `ans2`〜`ans10` — N 計算前の結果（最大 10 段）

### ユーザ変数

- 書式: `x = 3` （半角空白なしでも可）
- セッション中のみ保持（電源を切ると消える）
- 予約名（pi, ans, sin, theme 等）は上書き不可

### 数値リテラル

- 10 進浮動小数: `1.5e-10` `3.14` `42`
- 16 進整数: `0xFF`（= 255）
- 2 進整数: `0b101`（= 5）
- 8 進整数: `0o777`（= 511）

### ページ末

- 右下に **`→ 次ページ`** または **`Page 1/2`** の表示
- 任意のキーで戻れる旨を小さく表示するか、Page 2 にまとめて記載

## 6. ページ 2 に載せる内容

タイトル: **`Psephos -- 関数電卓 ヘルプ (2/2)`**

### キー操作

| キー | 動作 |
|---|---|
| ↑ / ↓ | 履歴呼び出し / 編集中バッファ復元 |
| ← / → | カーソル左右移動 |
| Home / End | 行頭 / 行末ジャンプ |
| Backspace | カーソル直前の文字を削除 |
| Enter | 式を評価 |
| ESC | Psephos 終了 |

### コマンド

| 入力 | 動作 |
|---|---|
| `help` + Enter | このヘルプ画面 |
| `theme` + Enter | テーマ一覧表示 |
| `theme <名前>` + Enter | テーマ適用 |

テーマ一覧: `default` / `amber` / `green` / `cyan` / `invert`

### 設定ファイル

- 場所: `/sd/psephos_config.txt`
- 永続化される項目: テーマ、有効桁、履歴上限
- 起動時に自動ロード、テーマ変更時に自動保存

### ページ末

- 右下に **`Page 2/2`** または **`← 前ページ`** の表示
- 中央または下部に **`任意のキーで戻る`** の案内（一回り小さい文字で）

## 8. 避けてほしいこと

- 320×320 以外のキャンバスサイズ
- §3 で指定した 4 色以外の使用（中間色・グラデーション・半透明・影含む）
- アンチエイリアス（縁が中間色になる設定）
- 文字の縁取り（4 色量子化で潰れます）
- 極小フォント（10 px 未満）
- 横スクロールを前提とする長い行
- アイコン画像（罫線文字 ─ │ ┌ ┐ ┘ └ ╋ などで代用してください）

## 9. エクスポート形式

- **PNG**
- 320 × 320 px ちょうど
- **RGB モード**（"RGB"）または **インデックスカラー**（"P"）
- 画像内で使われている色は **§3 の 4 色だけ**（混色されないこと）
- ファイル名案: `help_ja_p1.png` / `help_ja_p2.png`

> 弊機側で受け取り後、PNG → 4-bit インデックス画像 → GS4_HMSB バイナリへ変換します。色は §3 の RGB と一致するピクセルを各ロール（BG/FG/DIM/ACC）にマッピングします。
